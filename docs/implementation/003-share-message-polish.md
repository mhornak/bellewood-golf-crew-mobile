# Iteration 003 — Polish the Share Message Format

**Date**: 2026-04-28
**Status**: In Progress

## Goal

Streamline the share-message text generated by `SessionCard.tsx`'s
`handleShareStatus` so it's quicker to read in iMessage. Drop emoji noise,
group players by status (In / Maybe / Out) with counts, tighten the date
format, and remove redundant labels.

## Background

The current message decorates every player line with a status emoji
(✅ / ❓ / ❌), uses `📊` and `📱` for stats and the link, and lists all
players in one undifferentiated block ordered by `filteredUsers`. With a
typical 8-player crew the result is visually busy and the most useful
piece of info — _how many are confirmed_ — is buried near the bottom of
the message.

Iteration 002 already moved the link itself to a rich preview tile, so
the inline `📱 Update your status:` label is now redundant on iOS and
adds nothing on Android (the URL is self-explanatory).

## What We Set Out to Build

`golf-scheduler-mobile/src/components/SessionCard.tsx::handleShareStatus`
rewritten to produce this format:

```
⛳ {Session Title}
{EEE, MMM d · h:mm a}

In ({inCount}){celebration-suffix}
  {nickname} {🚶|🛺}{ · {note}}
  …

Maybe ({maybeCount})
  {nickname}, {nickname}, …

Out ({outCount})
  {nickname}, {nickname}, …

{Universal Link URL}
```

The ⛳ on the title line and the 🚶/🛺 transport emojis on IN players
were restored mid-iteration after the all-text version felt sterile in
real-device testing. They're the only two non-celebration emojis kept;
both carry information (golf identity at the top, transport mode per
confirmed player) rather than decoration.

## Design Decisions

These were settled with Mark before coding:

- **Maybe / Out lists are comma-separated.** Saves vertical space; soft
  data anyway.
- **Celebration line stays** but is appended to the `In` header instead
  of being its own emoji-prefixed line:
  - `In (8) — 🎯 2 foursomes ready`
  - `In (6) — ⛳ 2 threesomes`
  - `In (4) — 🏌️ 1 foursome`
- **Tag line dropped entirely.** No `🏷️ Bellewood Crew`. Subgroup context
  is carried by who's in the iMessage thread.
- **Creator line dropped.** "Created by MHo" reading in a message _from_
  MHo is redundant.
- **Date tightened** from `April 28th, 2026 2:00 PM` to
  `Tue, Apr 28 · 2:00 PM` (drop year; day-of-week up front).
- **In players keep one line each** with transport emoji (🚶 / 🛺) and
  any note appended. Transport text was tried first ("walking"/"riding")
  and felt sterile; emojis carry the same info more compactly.
- **Per-player status emojis dropped** (✅ / ❓ / ❌). Section header
  carries the status; per-line status emoji was the worst offender for
  visual noise in the old format.
- **⛳ added to the title line.** One golf-themed character at the top
  signals "this is a golf message" without weighing the body down.
- **URL on its own line, no label.** `📱 Update your status: <url>` →
  bare URL on a blank-line-separated line.

## Architecture Notes

- Pure mobile change. No web changes, no AWS changes, no rebuild required.
- Verifiable via Expo Go / dev client — no native entitlement changes,
  no need to push to TestFlight just to test.
- Date formatting uses `date-fns` `format()` with the format string
  `"EEE, MMM d '·' h:mm a"` (middle dot escaped as a literal).
- Player grouping derives directly from `golfUtils.findUserResponse`'s
  `status` field; no new helper needed.
